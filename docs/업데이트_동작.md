# 자동 업데이트 동작 — 무엇을 바꾸고, 의존성은 어떻게 챙기나

> 자주 나온 질문 세 가지에 답한다: (1) 자동 업데이트는 어떤 식으로 업데이트하나,
> (2) `scripts/run_this_before.py` 도 바뀌면 같이 업데이트되나, (3) 새 패키지가
> 추가됐을 때 의존성은 어떻게 갱신하나.

관련 코드: `aoi_verification/app/utils/updater.py` ·
`aoi_verification/app/ui/main_window.py`(`_on_update_found`/`_on_update_applied`).

---

## 1. 언제·어떻게 업데이트되나

- **확인 시점**: 앱 시작 직후 백그라운드로 한 번(조용히), 그리고 셋업 화면의
  **[업데이트 확인]** 버튼으로 수동.
- **비교 방법**: 동봉된 `app/VERSION`(JSON: `{sha, branch, repo}`)의 커밋 SHA 를
  GitHub 의 브랜치 HEAD 와 비교한다. 다르면 '업데이트 있음' 팝업.
- **받는 것**: 동의하면 브랜치 zip 을 받아 **앱 구동에 필요한 것을 전부** 앱 폴더로
  **미러링**한다(새 모듈·리소스·문서·스크립트까지 — 누락 없음). 진행은 로딩바에 단계별
  (다운로드 → 압축 해제 → 적용)로 표시된다.
- **적용 후**: 안내 후 **프로그램을 자동 종료**한다(자동 재실행 안 함 — 사용자가 다시 실행).
- **개발/소스(git) 실행에서는 자동 적용을 하지 않는다**(`is_git_checkout` 가드).
  로컬 변경이 날아가지 않도록, 이 경우엔 `git pull` 안내만 띄운다. 즉 **자동 적용은
  포터블 빌드에서만** 일어난다.

## 2. 무엇이 바뀌고, 무엇은 안 바뀌나

| 항목 | 자동 업데이트로 바뀌나 |
|---|---|
| `aoi_verification/`(앱 코드) | **예** |
| `main.py` · `requirements.txt` | **예** |
| `docs/` · `scripts/` · 기타 구동에 필요한 파일 | **예**(전부 미러링) |
| `dev/`(개발 전용 모음: `tests/`·`bench결과/`·`양식.xlsx`) | **아니오**(제외) — 단 `dev/양식.xlsx` 만 앱 루트로 따로 복사 |
| `python/`(번들 런타임) | **아니오** |
| 설치된 패키지(torch·openvino …) | **아니오 — 다시 설치하지 않음** |

> 제외 목록은 `updater._UPDATE_SKIP_TOP` 에서 관리한다. 구동에 필요한 새 폴더/파일은
> 별도 작업 없이 자동 포함되고, `scripts/run_this_before.py` 도 함께 받아둔다(실행은 수동).

즉 **자동 업데이트는 코드만 바꾸고 의존성 패키지는 건드리지 않는다.** 무거운 런타임을
다시 받지 않아 빠르고 안전하지만, **새 패키지가 필요한 변경은 자동으로 들어오지 않는다.**

## 3. `run_this_before.py` 는 자동 업데이트되나?

`scripts/run_this_before.py` 는 **처음 한 번** 환경을 준비하는(=`requirements.txt` 설치 +
모델 주머니 옵션 패키지 best-effort 설치) **개발용 준비 스크립트**다. 자동 업데이트는 이
파일을 최신본으로 **받아 둘 수는** 있지만(`_copy_run_this_before`), **자동으로 실행하진
않는다.** 따라서 의존성이 바뀌었다면 사용자가 한 번 갱신해 줘야 한다(아래 4).

## 4. 의존성이 바뀌었을 때 — 알림 + 수동 갱신

업데이트가 `requirements.txt` 의 **변경을 감지하면**(`deps_changed()`), 종료 안내에
다음 문구를 덧붙여 알린다(자동 재설치는 하지 않음 — 회사망/권한 문제로 멈추지 않도록):

- **개발/소스(git) 사용자**: `git pull` 후 `python scripts/run_this_before.py` 재실행.
  (이미 설치된 패키지는 'Requirement already satisfied' 로 빠르게 넘어가 여러 번 실행해도 안전)
- **포터블 사용자**: 앱 폴더의 번들 파이썬으로
  `python\python.exe -m pip install -r requirements.txt` 를 한 번 실행하거나,
  새 포터블 빌드를 받는다.

> 처음 도입 시점(앱 폴더에 `requirements.txt` 가 아직 없던 상태)에는 '바뀜'으로
> 오인하지 않도록, **둘 다 있고 내용이 다를 때만** 알림을 띄운다.

## 5. 한 줄 요약

- 자동 업데이트 = **앱 코드만** 교체(포터블에서만 자동). 런타임·패키지는 그대로.
- `run_this_before.py`/의존성은 **자동 설치 안 됨** — 바뀌면 **알림**이 뜨고, 위 절차로
  **수동 갱신**한다.
