# 외부 도구(Spyder · Anaconda · conda) 탐지·조치 보고서

회사 보안 정책상 본 프로젝트는 **Spyder · Anaconda · Miniconda · conda** 와 어떤
형태로도 연관·설치되어선 안 된다.  사용자(king-taek) 가 자기 PC 에서 관련 파일을
발견해 추적·정리한 일련의 과정을 기록한다.

본 보고서 자체는 사고 기록 용도로 키워드를 그대로 포함하므로, 저장소 전체를 스캔
하는 가드 테스트(``tests/test_no_spyder_conda.py``) 의 화이트리스트에 등록되어 있다.

---

## 1. 배경

- 프로젝트: AOI 검증 — Windows 데스크톱(GUI, PyQt6) 도구.
- 정책: 사용자 PC 에 Spyder / Anaconda / Miniconda / conda 설치 또는 연관 금지.
- 계기: 사용자가 PC 점검 중 위 키워드가 포함된 파일/실행 파일을 발견.

## 3. 조사 결과

### 3.1 본 프로젝트의 영향 분석 (전수조사)

- **저장소 git 전체 이력 (`git log --all`)**: 위 파일들 중 어느 것도 본 저장소에서
  생성·커밋된 적 **없음**.
- **현재 워크트리 전수 스캔**: 본 저장소의 가드 파일(`tests/test_no_spyder_conda.py`,
  `scripts/internal/verify_no_forbidden.py`) 외에는 해당 키워드 0건.
- **의존성 그래프**(`requirements.txt` 직접 + 설치된 패키지 메타데이터):
  Spyder/conda/anaconda/miniconda 또는 그 transitive 의존이 **0건**.  pip 으로는
  애초에 conda 를 설치할 수 없음(별도 패키지 매니저).
- **빌드 스크립트**: `scripts/internal/build_windows.bat`, `scripts/internal/make_portable.bat`,
  `scripts/run_this_before.py` 모두 표준 `python -m venv` + `pip install` 또는
  python-build-standalone(자체 포함 CPython) 만 사용 — 외부 패키지 매니저 미사용.

→ **본 프로젝트가 위 항목들을 설치하거나 끌어들이지 않았음** 이 확인됨.

### 3.2 발견 파일별 정체

- **`spyder_kernels.json`** (Pylance 번들): Pylance 가 Jupyter 커널 사양을 인식하기
  위해 동봉한 **읽기 전용 메타데이터 JSON**.  실행 파일 아님, 설치 아님.
- **`create_conda.py`** (ms-python.python 번들): VS Code 명령 팔레트
  `Python: Create Environment` 흐름에서 사용자가 **"Conda" 옵션을 명시적으로 선택할
  때만** 호출되는 보조 스크립트.  확장 설치만으로는 conda 가 깔리지 않음.
- **`conda.pyi`** (Pylance 번들): 정적 분석용 **타입 스텁(텍스트)**.  실행/설치 아님.
- **`conda.exe`**: 출처는 사용자 측 추가 조사 사안. 가능성:
  (a) VS Code `Create Environment → Conda` 선택 시 자동으로 받아 깔리는 Miniconda,
  (b) 별도 인스톨러 수동 실행, (c) 다른 SDK 동봉.  본 프로젝트와 무관함은 확정.

- **테스트 가드**: 어떤 커밋이든 파일 내용·이름에 단어 경계 `(ana|mini)?conda` /

  들어오는 경우에도 즉시 발견.
- **워크스페이스 설정**: VS Code 의 환경 생성 실험성 안내 비활성 + 표준 인터프리터
  고정으로 외부 매니저 자동 설치 트리거를 줄임.

## 6. 권고

### 6.1 사내 IT 차원(가장 확실한 차단막)
- 방화벽/프록시에서 `repo.anaconda.com`, `repo.continuum.io` 호스트 차단 →
  VS Code 등이 외부 패키지 매니저 인스톨러를 받을 수 없게.
- AppLocker / 소프트웨어 제한 정책으로 `*Miniconda*Setup*.exe`, `*Anaconda*.exe`,
  `*conda.exe` 류 실행 차단 → 사용자가 실수로 실행해도 OS 단에서 막힘.

### 6.2 사용자 차원
- VS Code 의 `Python: Create Environment` 명령에서 외부 매니저 옵션을 **선택하지
  않기** — 항상 `Venv` 만 선택.  외부 매니저 옵션은 사용자가 명시적으로 클릭해야만
  자동 다운로드가 시작됨.
- 사용자 전역 `settings.json` 에 다음 추가(전역 보호):
  ```json
  {
    "python.experiments.enabled": false,
    "python.experiments.optOutFrom": ["All"]
  }
  ```
- 주기적으로 `where conda` / `where spyder` 점검(공란 유지 확인).

### 6.3 프로젝트 차원(이미 적용)
- 다층 가드(테스트 + 런타임 + 워크스페이스 설정) 가 모든 향후 커밋·빌드·열기 흐름에서
  자동으로 점검.  사람이 잊어도 자동 차단.

## 7. 결론

본 프로젝트가 금지 도구를 설치한 정황은 **없음**.  발견된 파일들은 (i) 본 프로젝트
밖의 Microsoft 확장 번들(정적 텍스트/스펙) 또는 (ii) 별도 인스톨러로 추정되며, 사용자가
실행 형태(`conda.exe`)를 직접 정리해 현 시점 공란 상태.  앞으로의 재발 가능성은 본
보고서 §4.1 의 코드 가드와 §6.1 의 IT 차원 통제로 다중으로 막힘.
