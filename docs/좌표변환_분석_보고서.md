# Camtek-KLA 좌표변환 분석 보고서

> 이 문서는 AI 에이전트가 본 코드베이스의 좌표 매칭 시스템을 이해하기 위한 기술 참조 문서다.
> 사람이 아닌 AI가 읽는 것을 전제로 작성되었으며, 모든 수식·상수·파일 경로·코드 참조를 명시한다.

---

## 1. 시스템 개요

이 애플리케이션은 반도체 AOI(Automated Optical Inspection) 이미지 매칭 검증 데스크톱 앱이다.
두 대의 서로 다른 AOI 장비(예: Camtek, KLA)가 같은 웨이퍼를 검사했을 때,
**같은 결함을 찍은 이미지끼리 짝을 짓는 것(매칭)**이 핵심 기능이다.

좌표 매칭 모드에서는 이미지 유사도 대신 **결함의 물리적 좌표(die 인덱스 + die 내부 위치)**를 비교해 매칭한다.
이를 위해 Camtek과 KLA 두 장비의 서로 다른 좌표계를 **하나의 통일된 좌표계(DefectCoord)**로 변환해야 한다.

### 1.1 좌표 매칭이 필요한 이유

- Camtek과 KLA는 같은 웨이퍼의 같은 결함을 다른 좌표 체계로 기록한다.
- 이미지 유사도 매칭은 느리고(CPU/GPU 리소스 필요), 좌표 매칭은 빠르다(수학 연산만).
- 좌표가 정확하면 매칭 정확도도 높다.

### 1.2 대상 장치

현재 모든 변환 상수는 **TB500** 패키지(반도체 제품) 기준이다.
다른 제품을 지원하려면 `models.py`의 상수를 제품별로 분기해야 한다.

## 3. 데이터 모델

### 3.1 DefectCoord (통일 좌표)

**파일**: `aoi_verification/app/coords/models.py` (라인 14-25)

```python
@dataclass(frozen=True)
class DefectCoord:
    col: int                          # 표준화된 die column 인덱스
    row: int                          # 표준화된 die row 인덱스
    x: float                          # die 내부 local X (µm)
    y: float                          # die 내부 local Y (µm)
    source: str                       # "camtek_ini" | "camtek_live" | "kla"
    native_x: Optional[float] = None  # KLA 원본 XREL (source="kla"일 때만)
    native_y: Optional[float] = None  # KLA 원본 YREL (source="kla"일 때만)
```

**필드 의미**:

| 필드 | 단위 | 설명 |
|------|------|------|
| `col` | 정수 | 웨이퍼 위 die의 X 방향 인덱스 (0-based, Camtek/KLA 통일) |
| `row` | 정수 | 웨이퍼 위 die의 Y 방향 인덱스 (0-based, Camtek/KLA 통일) |
| `x` | µm | die 내부에서 결함의 X 좌표 (좌측 기준) |
| `y` | µm | die 내부에서 결함의 Y 좌표 (하단 기준) |
| `source` | 문자열 | 좌표를 추출한 소스 파서 식별자 |
| `native_x` | µm | KLA 원본 XREL 값 (보고서 출력용) |
| `native_y` | µm | KLA 원본 YREL 값 (보고서 출력용) |

### 3.2 TB500 변환 상수

**파일**: `aoi_verification/app/coords/models.py` (라인 44-60)

#### Camtek INI 변환 상수

| 상수명 | 값 | 단위 | 설명 |
|--------|-----|------|------|
| `CAMTEK_PITCH_X` | 37247.7 | µm/die | X 방향 die 폭 (≈ 37.2mm) |
| `CAMTEK_PITCH_Y` | 44905.4 | µm/die | Y 방향 die 높이 (≈ 44.9mm) |
| `CAMTEK_COL_OFFSET` | 2 | 정수 | Camtek Col → 표준 col 변환 오프셋 |
| `CAMTEK_ROW_TOTAL` | 7 | 정수 | Camtek Row → 표준 row 변환 (행 반전) |

#### KLA .001 변환 상수

| 상수명 | 값 | 단위 | 설명 |
|--------|-----|------|------|
| `KLA_ZERO_X` | 3 | 정수 | XINDEX → 표준 col 변환 오프셋 (패키지 X수 7의 절반) |
| `KLA_ZERO_Y` | 3 | 정수 | YINDEX → 표준 row 변환 오프셋 (패키지 Y수 6의 절반) |

## 4. 좌표 소스 파서

### 4.1 우선순위 체인

**파일**: `aoi_verification/app/coords/__init__.py`

좌표 추출 시 세 파서를 순서대로 시도하고, **첫 번째로 성공한 결과를 반환**한다:

```
1순위: camtek_live (LIVE 파일명 파서)    → 파일명만으로 즉시 파싱
2순위: camtek_ini (Camtek INI 파서)     → 같은 폴더의 INI 파일 참조
3순위: kla_info (KLA .001 파서)         → 같은 폴더의 .001 파일 참조
```

```python
def resolve(image_path: Path) -> Optional[DefectCoord]:
    c = camtek_live.resolve(path)      # 1순위
    if c is not None: return c
    c = camtek_ini.resolve(path)       # 2순위
    if c is not None: return c
    c = kla_info.resolve(path)         # 3순위
    if c is not None: return c
    return None
```

`resolve_batch(paths)`는 여러 경로를 루프로 처리한다.
LRU 캐시 덕분에 같은 폴더의 INI/KLA 파일은 한 번만 읽는다.

### 4.2 Camtek LIVE 파일명 파서

**파일**: `aoi_verification/app/coords/camtek_live.py`

LIVE 형식 파일명에서 col, row, x, y를 **직접 추출**한다.
파일명 자체에 die 좌표가 인코딩되어 있으므로 별도 메타데이터 파일이 불필요하다.

#### 지원하는 파일명 형식

**형식 A** (x/y 뒤에 DefectName 선택):
```
R_TB500_LIVE_PI4_PXU-PIDS3_00RMF043XYE0_5_3_21620.211_7230.807_Foreign Material.jpg
                                          ^col ^row ^x         ^y
```

**형식 B** (DefectName 다음에 x/y):
```
R_TB500_LIVE_PI4_VLP-PDIS3_W6317098XYB5_4_5_Over Sized Bump_30229.803_1987.994.jpg
                                         ^col ^row                    ^x        ^y
```

#### 정규표현식

```python
# 형식 A: _col_row_x[.xx]_y[.yy][_DefectName]
_PAT_A = re.compile(r'_(\d+)_(\d+)_([\d]+(?:\.[\d]+)?)_([\d]+(?:\.[\d]+)?)(?:_.+)?$')

# 형식 B: _col_row_DefectName_x[.xx]_y[.yy]
_PAT_B = re.compile(r'_(\d+)_(\d+)_.+_([\d]+(?:\.[\d]+)?)_([\d]+(?:\.[\d]+)?)$')
```

#### 변환

LIVE 파일명의 col, row, x, y는 **이미 변환 완료된 값**이므로 추가 변환 없이 그대로 사용한다:

```
col = 파일명에서 추출한 정수 (1번째 캡처 그룹)
row = 파일명에서 추출한 정수 (2번째 캡처 그룹)
x   = 파일명에서 추출한 실수 (3번째 캡처 그룹) — die 내부 X (µm)
y   = 파일명에서 추출한 실수 (4번째 캡처 그룹) — die 내부 Y (µm)
source = "camtek_live"
```

#### KLA 파일명 오매칭 방지

KLA 이미지 파일명(예: `W6459076XYG1_2_0_23_2.jpg`)도 `_PAT_A` 정규식에 매칭될 수 있다.
이를 방지하기 위해 **매칭 전 접두부에 `_`가 있는지 검증**한다:

```python
def resolve(image_path: Path) -> Optional[DefectCoord]:
    stem = image_path.stem
    for pat in (_PAT_A, _PAT_B):
        m = pat.search(stem)
        if m and '_' in stem[:m.start()]:   # ← 접두부 _ 검증
            return _extract(m)
    return None
```

**판별 원리**:
- LIVE 파일명: `R_TB500_LIVE_..._5_3_21620.211_7230.807` → 접두부 `R_TB500_LIVE_...`에 `_` 있음 → 통과
- KLA 파일명: `W6459076XYG1_2_0_23_2` → 접두부 `W6459076XYG1`에 `_` 없음 → 거부
- Camtek INI 파일명: `272649.165672.c.1796377335.1` → 정규식 자체가 안 맞음 → None

### 4.3 Camtek INI 파서

**파일**: `aoi_verification/app/coords/camtek_ini.py`

같은 폴더의 `ColorImageGrabingInfo.ini` 파일을 읽어 좌표를 추출한다.

#### INI 파일 구조

```ini
[272646.165679.c.1000203959.2.jpeg]
X=272647.761085349          ← 절대 wafer X 좌표 (µm)
Y=165675.853620774          ← 절대 wafer Y 좌표 (µm)
Col=7                       ← Camtek die column (1-based, Camtek 고유 인덱싱)
Row=3                       ← Camtek die row (1-based, Camtek 고유 인덱싱)
FaultX=272647.761085349     ← X의 대체값 (X가 없을 때 폴백)
FaultY=165675.853620774     ← Y의 대체값 (Y가 없을 때 폴백)
PixelSizeX=0.172499999403954
PixelSizeY=0.172499999403954
Mag=20
...
```

- 섹션명 `[파일명.jpeg]`으로 이미지별 좌표를 구분한다.
- 필수 키: `X` (또는 `FaultX`), `Y` (또는 `FaultY`), `Col`, `Row`
- 하나라도 없으면 해당 이미지의 좌표는 `None`을 반환한다.

#### INI 파일 탐색

```python
_INI_CANDIDATES = ("ColorImageGrabingInfo.ini", "ColorImageGrabinginfo.ini")
```

이미지와 같은 폴더에서 위 이름의 파일을 찾는다. 대소문자 두 가지를 시도한다.

#### 변환 공식

```
col = Col - CAMTEK_COL_OFFSET         = Col - 2
row = CAMTEK_ROW_TOTAL - Row          = 7 - Row
x   = X - Col × CAMTEK_PITCH_X       = X - Col × 37247.7
y   = Y - Row × CAMTEK_PITCH_Y       = Y - Row × 44905.4
source = "camtek_ini"
```

**변환의 의미**:
- `Col - 2`: Camtek의 1-based 인덱스를 0-based 표준 인덱스로 변환 (오프셋 2)
- `7 - Row`: Camtek은 Row가 위에서 아래로 증가하는데, 표준은 아래에서 위로 증가 → 반전
- `X - Col × 37247.7`: 절대 좌표에서 die 원점을 빼서 die 내부 좌표로 변환
- `Y - Row × 44905.4`: 마찬가지로 Y 방향 die 내부 좌표로 변환

#### 코드

```python
def _extract_coord(content: str) -> Optional[DefectCoord]:
    # ... 키-값 파싱 생략 ...
    X = fget("X") if fget("X") is not None else fget("FAULTX")
    Y = fget("Y") if fget("Y") is not None else fget("FAULTY")
    Col = fget("COL")
    Row = fget("ROW")
    if None in (X, Y, Col, Row):
        return None
    col_i = int(Col)
    row_i = int(Row)
    col = col_i - CAMTEK_COL_OFFSET          # 표준 die index
    row = CAMTEK_ROW_TOTAL - row_i            # 행 반전
    x = X - col_i * CAMTEK_PITCH_X           # die-내부 x
    y = Y - row_i * CAMTEK_PITCH_Y           # die-내부 y
    return DefectCoord(col=col, row=row, x=x, y=y, source="camtek_ini")
```

### 4.4 KLA .001 파서

**파일**: `aoi_verification/app/coords/kla_info.py`

같은 폴더의 `.001` 확장자 파일(KLA 검사 결과 파일)을 읽어 좌표를 추출한다.

#### .001 파일 구조

```
FileVersion 1 2;
InspectionStationID "KLA" "LDS CIRCL" "K3";
LotID "TB500INT.271@6324";
WaferID "W6459076XYG1";
DiePitch 3.7247930000e+004 4.4905340000e+004;    ← DiePitchX, DiePitchY (µm)
DieOrigin 0.0000000000e+000 0.0000000000e+000;
DefectRecordSpec 22 DEFECTID X Y XREL YREL XINDEX YINDEX ...;
TiffFileName W6459076XYG1_2_0_23_2.jpg;          ← 이미지 파일명
DefectList
 2 67855.280 14093.720 11819.421 13870.779 2 0 ...;  ← 결함 데이터
```

**DefectList 필드 순서** (0-based, 공백 구분):

| 인덱스 | 필드명 | 설명 |
|--------|--------|------|
| 0 | DEFECTID | 결함 ID |
| 1 | X | 절대 wafer X (µm) |
| 2 | Y | 절대 wafer Y (µm) |
| 3 | XREL | die 내부 상대 X (µm) |
| 4 | YREL | die 내부 상대 Y (µm) |
| 5 | XINDEX | die X 인덱스 (부호 있는 정수, 중심 기준) |
| 6 | YINDEX | die Y 인덱스 (부호 있는 정수, 중심 기준) |

#### KLA 파일명 패턴

```
W6459076XYG1_2_0_23_2.jpg
^WaferID     ^XINDEX ^YINDEX ^DEFECTID ^TEST
```

- WaferID: 웨이퍼 식별자
- XINDEX, YINDEX: die 인덱스 (중심 die를 0으로 하는 부호 있는 정수)
- DEFECTID, TEST: 결함 번호와 검사 번호

#### INFO 파일 탐색

```python
def _find_info_file(folder: Path) -> Optional[Path]:
    # 1순위: .001 확장자
    for p in folder.glob("*.001"):
        return p
    # 2순위: .jpg/.jpeg/.pass/.ini/.py가 아닌 유일한 파일
    candidates = [p for p in folder.iterdir()
                  if p.is_file() and p.suffix.lower() not in ('.jpg', '.jpeg', '.pass', '.ini', '.py')
                  and not p.name.startswith('.')]
    if len(candidates) == 1:
        return candidates[0]
    return None
```

#### 변환 공식

```
col = XINDEX + KLA_ZERO_X     = XINDEX + 3
row = YINDEX + KLA_ZERO_Y     = YINDEX + 3
x   = round(XREL)
y   = round(DiePitchY - YREL)
source = "kla"
native_x = XREL               (원본 보존)
native_y = YREL               (원본 보존)
```

**변환의 의미**:
- `XINDEX + 3`: KLA는 중심 die를 0으로 하는 부호 있는 인덱스. +3으로 0-based 양수 인덱스로 변환
- `YINDEX + 3`: Y 방향도 마찬가지
- `round(XREL)`: die 내부 상대 X를 정수 µm로 반올림
- `round(DiePitchY - YREL)`: KLA의 Y축 원점이 die 상단이므로, DiePitchY에서 빼서 하단 기준으로 변환

#### 코드

```python
@lru_cache(maxsize=256)
def load_folder(folder: Path) -> dict[str, DefectCoord]:
    result: dict[str, DefectCoord] = {}
    for stem, r in load_folder_raw(folder).items():
        col = r["xindex"] + KLA_ZERO_X
        row = r["yindex"] + KLA_ZERO_Y
        x = round(r["xrel"])
        y = round(r["die_pitch_y"] - r["yrel"])
        result[stem] = DefectCoord(
            col=col, row=row, x=float(x), y=float(y), source="kla",
            native_x=float(r["xrel"]), native_y=float(r["yrel"]),
        )
    return result
```

## 6. 좌표 매칭 엔진

**파일**: `aoi_verification/app/workers/coord_matcher.py`

### 6.1 매칭 알고리즘 개요

1. 모든 ref/val 이미지의 좌표를 일괄 프리패치한다 (`resolve_batch`).
2. val 이미지들을 `(col, row) → [(path, x, y), ...]` 맵으로 구성한다.
3. 각 ref에 대해 **(col, row) ±1 이웃 9칸**의 val 후보를 모은다.
4. 후보와의 유클리드 거리가 `tolerance × 3` 이내인 것만 추린다.
5. `_select_coord_candidates()`로 최종 후보를 선택하고 score로 인코딩한다.

### 6.2 (col, row) ±1 이웃 게이트

**참조 근거**: AOI Data Viewer VBA `Module_Compare.AOIMapCompare`에서
`Abs(col차)<=1 And Abs(row차)<=1`로 ±1을 허용한다.

```python
for dc in (-1, 0, 1):
    for dr in (-1, 0, 1):
        for vpath, vx, vy in val_coord_map.get((ref_col + dc, ref_row + dr), ()):
            dist = math.hypot(ref_x - vx, ref_y - vy)
            if dist <= tol3:
                within3.append((vpath, dist))
```

이 루프는 ref의 (col, row)를 중심으로 3×3 = 9개 칸을 탐색한다.
`dc` = -1, 0, 1 / `dr` = -1, 0, 1 → 총 9개 조합.

**왜 ±1이 필요한가**:
- KLA와 Camtek의 die 인덱스 변환 공식이 다르므로 같은 물리적 die도 인덱스가 1 차이날 수 있다.
- 위 예시 3건 모두 row가 1 차이. 정확 일치만 하면 매칭이 전멸한다.

### 6.3 거리 계산

유클리드 거리:
```
dist = √((ref_x - val_x)² + (ref_y - val_y)²)
```

x, y 모두 µm 단위이므로 dist도 µm 단위이다.

### 6.4 후보 선택 규칙

**파일**: `coord_matcher.py`의 `_select_coord_candidates()` 함수

```python
CONFIDENT_DIST = 20.0  # µm

def _select_coord_candidates(within3, tol):
    ordered = sorted(within3, key=lambda x: x[1])  # 거리 오름차순
    if not ordered:
        return []
    if ordered[0][1] <= CONFIDENT_DIST:  # 최소 거리 ≤ 20 µm
        ordered = ordered[:1]            # 가장 가까운 1장만
    return [(p, score_of(d)) for p, d in ordered]
```

| 조건 | 동작 |
|------|------|
| 최소 거리 ≤ 20 µm | **확신 높음** — 가장 가까운 1장만 반환 |
| 최소 거리 > 20 µm | **확신 낮음** — `tol × 3` 이내 모든 후보를 반환 |

### 6.5 Score 인코딩

거리를 `-∞ ~ 1.0` 범위의 score 값으로 인코딩한다:

```python
def score_of(dist):
    if dist <= tol:
        return max(0.0, 1.0 - dist / tol)   # 양수: 허용 범위 내
    return -(dist / tol)                     # 음수: 허용 범위 초과
```

| 거리 범위 | Score 범위 | 의미 |
|-----------|-----------|------|
| dist = 0 | score = 1.0 | 완벽 일치 |
| 0 < dist ≤ tol | 0.0 < score < 1.0 | 허용 범위 내 (가까울수록 1에 가까움) |
| dist = tol | score = 0.0 | 허용 범위 경계 |
| tol < dist ≤ 3×tol | -3.0 < score < 0.0 | 허용 범위 초과 (멀수록 음수 커짐) |
| dist > 3×tol | 후보 제외 | 매치 실패 |

**Score 역산** (UI에서 거리 복원):
```
score ≥ 0: dist = (1 - score) × tolerance
score < 0: dist = (-score) × tolerance
```

### 6.6 허용 오차 설정

| 설정 | 값 | 위치 |
|------|-----|------|
| 기본값 | 500.0 µm | `config.py` → `SimilarityConfig.coord_tolerance` |
| UI 슬라이더 최소 | 10.0 µm | `config.py` → `Matching.COORD_TOL_MIN` |
| UI 슬라이더 최대 | 5000.0 µm | `config.py` → `Matching.COORD_TOL_MAX` |

### 6.7 매치 실패 처리

`_match_neighbors()`가 빈 리스트를 반환하면(모든 이웃의 거리 > 3×tol),
해당 ref는 `failed_set`에 추가된다.

```python
result = _match_neighbors(ref_coord.x, ref_coord.y, ref_coord.col, ref_coord.row,
                          val_coord_map, tol)
self._results[(slot, ref.path)] = result
if not result:
    self.failed_set.add((slot, ref.path))
```

좌표가 아예 없는 ref는 **클래식 유사도 매칭**으로 폴백한다:
```python
if ref_coord is None:
    fallback_refs.append(ref)
# ...
for ref in fallback_refs:
    cands = score_ref_classical(ref, vals, threshold=0.0, cfg=self._cfg, ...)
    self._results[(slot, ref.path)] = [(c.item.path, float(c.score)) for c in cands]
```

### 6.8 CoordScheduler 전체 흐름

```
CoordScheduler.run()
  │
  ├─ 1. 좌표 일괄 프리패치: resolve_batch(all_paths) → coord_cache
  │
  ├─ 2. 좌표 없음 조기 종료: 모든 ref에 좌표 없으면 에러
  │
  └─ 3. 슬롯별 루프:
       │
       ├─ val → (col,row) 맵 구성
       │
       ├─ ref별 좌표 매칭:
       │   ├─ 좌표 있음 → _match_neighbors() → results에 저장
       │   └─ 좌표 없음 → fallback_refs에 추가
       │
       └─ fallback_refs → score_ref_classical() (이미지 유사도)
```

def _format_score(self, score: float) -> str:
    if self._coord_mode:
        if score >= 0:
            dist = (1.0 - score) * self._tolerance
            return f"{dist:.0f} µm"                    # 예: "121 µm"

## 10. 좌표계 비교 요약

### 10.1 Camtek 좌표계

```
원본 필드: X, Y (절대 wafer 좌표, µm), Col, Row (die 인덱스, 1-based)

 Col →  2  3  4  5  6  7   (Camtek 원본)
        ↓  ↓  ↓  ↓  ↓  ↓
 col →  0  1  2  3  4  5   (표준, Col - 2)

 Row →  7  6  5  4  3  2  1  0   (Camtek 원본, 위에서 아래)
        ↓  ↓  ↓  ↓  ↓  ↓  ↓  ↓
 row →  0  1  2  3  4  5  6  7   (표준, 7 - Row, 아래에서 위)
```

### 10.2 KLA 좌표계

```
원본 필드: XREL, YREL (die 내부 상대 좌표, µm), XINDEX, YINDEX (die 인덱스, 중심 0)
           DiePitchX, DiePitchY (die 크기, µm)

 XINDEX → -3 -2 -1  0  1  2  3   (KLA 원본, 중심 0)
           ↓  ↓  ↓  ↓  ↓  ↓  ↓
 col →     0  1  2  3  4  5  6   (표준, XINDEX + 3)

 YINDEX → -3 -2 -1  0  1  2      (KLA 원본, 중심 0)
           ↓  ↓  ↓  ↓  ↓  ↓
 row →     0  1  2  3  4  5      (표준, YINDEX + 3)

 Y 좌표: YREL = die 상단으로부터의 거리
         y = DiePitchY - YREL    (die 하단으로부터의 거리로 변환)
```

### 10.3 Camtek ↔ KLA 좌표 정렬 결과

두 변환을 거치면 같은 결함이 유사한 (col, row, x, y)를 갖게 되지만:
- **col, row**: ±1 범위 내에서 일치 (정확히 같지 않을 수 있음)
- **x, y**: 수십~수백 µm 차이 (변환 공식의 반올림, 장비 측정 오차)

이 차이를 수용하기 위해:
- (col, row) ±1 이웃 게이트
- 유클리드 거리 허용 오차 (기본 500 µm)

## 11. 알려진 주의사항

### 11.1 장치 특화 상수

모든 변환 상수(PITCH, OFFSET, ZERO)는 **TB500 제품 전용**이다.
다른 제품의 웨이퍼를 처리하려면 `models.py`의 상수를 제품별로 분기해야 한다.

### 11.2 DiePitch 불일치

Camtek 상수(`CAMTEK_PITCH_X = 37247.7`)와 KLA INFO 파일의 DiePitch(`3.7247930000e+004 = 37247.93`)는
소수점 이하에서 미세하게 다르다 (0.23 µm 차이). 이 차이는 매칭에 유의미한 영향을 주지 않지만,
변환 정확도를 높이려면 향후 통일할 수 있다.

### 11.3 LRU 캐시

- `camtek_ini.load_folder`: maxsize=256
- `kla_info.load_folder`: maxsize=256
- `kla_info.load_folder_raw`: maxsize=256

같은 폴더의 여러 이미지를 처리할 때 INI/INFO 파일을 한 번만 읽는다.
새 폴더 데이터를 테스트할 때는 `cache_clear()`가 필요하다.

### 11.4 매치 실패 검토의 점수 체계

매치 실패 검토 다이얼로그(`UnmatchedReviewDialog`)는 좌표가 아닌 **이미지 유사도**로 후보를 채점한다.
따라서 좌표 모드에서도 이 다이얼로그의 점수는 유사도 백분율(%)로 표시된다.
이는 의도적인 설계이다 — 좌표 매칭에 실패한 이미지는 좌표가 신뢰할 수 없으므로
시각적 유사도가 더 적절한 판단 기준이다.
