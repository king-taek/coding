### A1/A2·D1 을 전부 구현하지 않은 이유 (안전 우선)
- **A2**: 캐시 파일 `exists()/stat()` 메모이즈는 같은 세션에서 캐시가 *생성된 뒤*
  조회되는 흐름과 충돌해 "방금 만든 캐시를 없다고 오판"할 수 있다. 소스 이미지
  mtime 은 이미 `cache.memo_mtime` 으로 메모이즈됨. → 추가 이득 작고 리스크 큼.
- **D1 god-class 분해**: 순수 유지보수(기능 변화 0)인데 위젯 부모/순서 회귀가
  단위 테스트로 검출되지 않는다. "버그 금지" 원칙상 별도 집중 작업으로 분리 권장.

## §2. 후속 과제 (미구현 — 우선순위·효과 정리)

### 성능 (P)
- **P1. 캐시 stat/exists 비동기·안전 메모이즈** — 현행: NAS 에서 캐시 확인마다
  동기 `stat()`(수천 회). 개선: 세션 중 *생성 시점에 무효화*하는 안전한 메모이즈
  또는 점수 캐시 비동기 로드. 효과: NAS 준비단계 수백 ms~수초 단축. (정합성 설계 필요)
- **P2. ORB 중심가중치 사전계산** — 현행: 중심가중 모드에서 (ref,val) 쌍마다
  좌표→거리 재계산. 개선: Feature 추출 시 1회 계산·저장, 비교 시 스케일만.
  효과: 중심가중 ORB 스코어링 부하 대폭↓(정확도 불변).
- **P3. SSIM 정준 크기 정규화** — 현행: shape 불일치 시 비교마다 리사이즈.
  개선: 추출 시 ROI 를 256² 로 1회 정규화. 효과: SSIM 경로 소폭↑.
  ⚠️ 정규화가 점수에 영향 줄 수 있어 정확도 검증 필수.

### 안정성 (R)
- **R1. except 일괄 audit** — 현행: `except Exception` ~191곳(그중 침묵 ~70곳).
  개선: 의도별 분류(옵션의존성 DEBUG / I·O WARNING / 그외 ERROR+트레이스백),
  lint 로 bare except 금지. 효과: 실제 버그 가시화, 지원비용↓. (§1-B1 의 확대판)
- **R2. 전역 가변 상태 보호** — 현행: `dev/benchmark.py _ENSEMBLE_REF_B` 등 전역
  락 없음(현재 순차라 안전). 개선: 파라미터화 또는 락 + "thread-safe 아님" 명시.
  효과: 향후 병렬화 안전.

### UX (U)
- **U1. 스캔 단계 취소/타임아웃** — 현행: 폴더 스캔은 동기 호출이라 중단 불가.
  개선: 취소 플래그를 진행 콜백에서 확인해 조기 종료(부분 결과 폐기). 효과: 느린 NAS 에서 탈출.
- **U2. 일괄 동작 확인 + 원자적 undo** — 현행: 선택모드 일괄 검증/제외 즉시 적용,
  undo 는 단일 결정만. 개선: 10장 초과 확인 다이얼로그 + 일괄을 history 1엔트리로.
- **U3. 세션 재개 UI 상태 복원 확대** — 현행: 폴더/임계치만 복원(스플리터 일부).
  개선: 스크롤·줌 슬라이더·스플리터 전부 세션 저장·복원.
- **U4. 오류의 사용자 가시화** — 현행: 로드 실패는 로그만. 개선: 실패 썸네일 배지 +
  배치 종료 시 실패 요약.
- **U5. 위젯 일관성 통합** — zoom_window/side_by_side_viewer/match_expand_view 의
  종료 라벨·크기 슬라이더·카운터 표준화.
- **U6. 대량 후보 탐색** — Stage 1 좌측 패널에 슬롯 점프/파일명 필터(현재 스크롤만).
- **U7. 키보드 접근성** — 선택모드 다이얼로그 Tab/Space/Ctrl+A 네비게이션.

### 유지보수 (A)
- **A1. god-class 분해** — main_window(1500+줄)에서 `UpdateManager`/`SessionManager`/
  `SystemMonitor` 분리, `match_page._build` 를 pane 별 헬퍼로. 효과: 테스트·확장성↑.
  ⚠️ 레이아웃 회귀가 자동 테스트로 안 잡혀 별도 집중 작업 + 수동 검증 권장.
- **A2. 거대 메서드 분할** — `dev/benchmark.run_recipe`(240줄), `match_page._build`(170줄),
  `match_page.load_state↔_start_precompute` 책임 분리.
- **A3. 중복 제거** — ZoomWindow 인스턴스화·패널 액션 처리·캐시 경로 로직 헬퍼화.

### 테스트·패키징 (T)
- **T1. UI 흐름 테스트** — pytest-qt 로 MatchPage 상태 전이/단축키 커버(현재 워커·순수함수 위주).
- **T2. 옵션 의존성 분리** — `requirements` base/full 분리(CPU-only 설치 경량화) + 미가용 시 안내 표시.
- **T3. root `tests/` 정리** — 로컬 빈 폴더(미추적). 혼란 방지를 위해 통합테스트 용도로 명시하거나 무시.

## §3. 탐색 중 제안되었으나 **이미 반영되어 불필요**한 항목 (정정)
- OpenVINO 전처리↔추론 **오버랩 이미 구현**(`embedder_openvino._preprocess_parallel`).
- 썸네일 **우선순위 큐**(`PRIORITY_ACTIVE_SLOT`, heapq)로 활성 슬롯 우선 — "첫 슬롯 30초 대기"는 과장.
- 특징 **디스크 캐시(.npz)** 로 재실행 시 미재계산 — 디코드 중복은 "첫 1회"로 한정.
- 점수쌍 병렬 처리는 **windowing**(`window=max(8,n_workers*4)`)으로 메모리 폭증 이미 방지.
- 선택모드 다이얼로그 **200장 페이지네이션**(직전 작업에서 추가됨).
- A1(추출 내 ROI 디코드 공유) — `pipeline.extract` 가 이미 1회 디코드 공유.
- NPU `batch=1` 은 동적 shape 미지원 회피용 **의도된 설정** — 상향은 호환성 위험.
