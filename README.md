# AOI 검증 프로그램

> 한국어 사용자 인터페이스(GUI)를 제공하는 AOI(자동 광학 검사) 검증 도구입니다.

- **개발자:** 임현택 (HyunTaek Lim)
- **소속:** Bump 2 Dept. / AOI Engineer
- **실행 환경:** **VS Code** 에서 가상환경(`venv`) 으로 실행. PyInstaller 호환은 유지.

---

## 1. 사전 준비

- Windows 10/11
- **VS Code** 와 **Python 확장**(Microsoft 공식) 설치
- 시스템에 **Python 3.10 이상**이 설치돼 있어야 함 ([python.org](https://www.python.org/downloads/) 에서 공식 설치본 사용).

## 2. 폰트 안내

한국어가 깨지지 않도록 다음 폰트 중 하나가 시스템에 설치되어 있어야 합니다.

- `Pretendard`
- `Noto Sans KR`
- `Malgun Gothic` (Windows 기본 — 보통은 이미 있음)

영문 타이틀은 `Orbitron`, 본문은 `Rajdhani` 를 우선 사용하며, 한글이 포함되면
한국어 폰트로 자동 폴백됩니다.

## 3. VS Code 에서 실행하기

### 3.1 프로젝트 폴더 열기

VS Code → `파일 → 폴더 열기` → 이 저장소 폴더(`coding-claude-aoi-verification-app-LAXpX`).

### 3.2 가상환경(`.venv`) 만들기

VS Code 안에서 **터미널 열기** (`Ctrl + ` ` 또는 메뉴 `Terminal → New Terminal`).
PowerShell 이 기본일 텐데 그대로 사용하면 됩니다.

```powershell
py -m venv .venv
# 또는
python -m venv .venv
```

### 3.3 venv 활성화

```powershell
.\.venv\Scripts\Activate.ps1
```

> **PowerShell 실행 정책 오류** 가 나오면 한 번만:
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```
> 그래도 안 되면 `cmd` 스타일로: `.\.venv\Scripts\activate.bat`

프롬프트 앞에 `(.venv)` 가 붙으면 성공.

### 3.4 패키지 설치

```powershell
python -m pip install --upgrade pip
pip install -r requirements.txt
```

### 3.5 (선택) Intel GPU 사용자 — OpenVINO 안내

Intel GPU 를 쓰는 환경에서는 실행 시 **OpenVINO 설치 안내** 가 뜰 수 있습니다.
이는 **선택 사항(고효율 모드 GPU 가속)** 이며 설치하지 않아도 모든 기본/검증
기능은 정상 동작합니다.

> **고효율 모드** 는 Intel GPU+OpenVINO 가 있으면 GPU 임베딩으로 후보를 빠르게
> 추리고, 없으면 CPU 고전 단독으로 자동 폴백합니다 (아래 5.1 참고).

### 3.5 VS Code 에 venv 등록

1. **Ctrl + Shift + P** → `Python: Select Interpreter`
2. 목록에서 **`.venv (Python 3.x.x)`** 또는 다음 경로 선택:
   ```
   .\.venv\Scripts\python.exe
   ```
3. 우측 하단 상태바에 `Python 3.x.x ('.venv')` 가 떠야 정상.

### 3.6 실행

세 가지 방법 — 편한 것 사용:

| 방법 | 동작 |
| --- | --- |
| `main.py` 열고 **F5** | 디버거로 실행 (중단점 사용 가능) |
| 우측 상단 **▶ Run Python File** | 디버거 없이 빠르게 실행 |
| 터미널에 `python main.py` | 가장 단순 |

`AOI 검증` 창이 뜨면 성공입니다.

> 메인 진입점인 `main.py` 는 `QApplication.instance()` 를 재사용하므로 IDE
> 가 두 번째 실행할 때도 안전합니다. 디버그를 중간에 강제 종료해도 다음
> 실행에 영향을 주지 않습니다.

### 3.7 포터블 배포(권장) — 파이썬 없는 PC 에 더블클릭 실행

파이썬이 설치되지 않은 Windows PC 에서도 실행되도록 **자체 포함 CPython 런타임**
을 폴더로 동봉합니다.  PyInstaller 부트로더가 회사 백신/보안 정책에 막혀 빌드·실행이
안 되는 경우를 피하고, **업데이트가 쉬운** 것이 장점입니다.

빌드는 **Windows + 인터넷** 환경에서 1회만:

```powershell
# 저장소 루트에서 (빌드/배포 스크립트는 scripts\ 폴더에 있습니다)
scripts\make_portable.bat
```

- 산출물 `dist_portable\` 구조:
  ```
  dist_portable\
    python\         ← 자체 포함 CPython + 의존성(torch/openvino 등) [무거움, 거의 불변]
    app\            ← aoi_verification 소스 + main.py + 양식.xlsx       [업데이트 대상]
    run_aoi.bat      ← 콘솔 없이 GUI 실행 (pythonw)
    run_aoi_debug.bat← 오류 진단용(콘솔 + traceback)
    update_app.bat   ← app\ 소스만 교체(git pull 또는 새 폴더 복사)
  ```
- **배포:** `dist_portable` 폴더 전체를 zip 으로 전달 → 대상 PC 에서 압축 해제 후
  **`run_aoi.bat` 더블클릭**(파이썬 불필요).
- **용량:** 고효율 모드(torch+openvino) 포함이라 폴더가 큽니다(대략 **1.3~2.0GB**,
  zip ≈ 0.6~1.0GB).
- **쉬운 업데이트:** 무거운 `python\` 은 그대로 두고 **작은 `app\` 소스만 교체**
  (`update_app.bat`) → 수 MB 만 갱신, 1.5GB 재배포 불필요.
- **앱이 안 켜질 때:** `run_aoi_debug.bat` 로 콘솔에 뜨는 오류(traceback)를 확인하세요.
- `scripts\make_portable.bat` 의 `PY_URL` 다운로드가 404 면, python-build-standalone
  [releases](https://github.com/astral-sh/python-build-standalone/releases) 에서 최신
  *install_only Windows x86_64* `.tar.gz` 링크로 `PY_URL` 을 교체 후 재실행하세요.

- 산출물: `dist\AOI_Verify\AOI_Verify.exe` (폴더 통째로 zip 배포). 설정은

- **첫 실행 경고:** 미서명 exe 는 SmartScreen/Defender 가 “알 수 없는 게시자” 로

## 4. 폴더 구조 가정

```
[기준 장비 최상위 폴더]
├── Slot_01/
│   ├── image1.jpeg
│   └── ...
├── Slot_02/
└── ...

[검증 장비 최상위 폴더]   ← 기준 장비와 동일한 Slot 명을 사용해야 매칭됩니다
├── Slot_01/
├── Slot_02/
└── ...
```

- 두 최상위 폴더는 **서로 다른 호기(장비)** 의 결과물입니다.
- 같은 경로를 두 번 입력하면 경고 모달이 표시됩니다.
- 한쪽에만 존재하는 Slot 은 사용자가 직접 매핑하거나 결과 보고서의
  `Slot 불일치 목록` 시트에 기록됩니다.

## 5. 사용 흐름 (요약)

1. **자동화 수준 선택** — `사진 직접 선택 + 매치는 자동` /
   `모든 사진 자동(그룹화로 Stage 1 건너뜀)`. 두 모드 모두 유사도를 한 번에
   계산한 뒤 자동으로 매칭하고, 사용자는 매치 검토 화면에서 확인합니다.
2. **유사도 엔진 모드 선택** (초기 화면) — 아래 5.1 참고
3. 기준·검증 장비 폴더와 호기 번호 입력, 유사도 임계치 설정 후 **`검증 시작`**
4. **Stage 1** — 후보 선별 (중앙 사진에 대해 `검증` 또는 `제외` 결정.
   `모든 사진 자동` 모드는 그룹화로 이 단계를 건너뜁니다)
5. **Stage 2** — 유사도 기반 자동 매칭
6. **매치 검토 → 매치 실패 사진 검토** 후 결과를 `양식.xlsx` 템플릿에 채워
   엑셀로 저장

### 5.1 유사도 엔진 모드

초기 화면에서 두 가지 중 하나를 선택합니다.

- **기본 모드 (정밀 비교)** — 모든 후보를 정밀 비교합니다. 정확하지만
  사진이 매우 많을 때 느립니다.
- **고효율 모드 (CPU+GPU)** — Intel GPU(MobileNetV3) 임베딩으로 후보를 빠르게
  추린 뒤, 상위 후보만 CPU 고전(pHash+ORB+SSIM)으로 정밀 재채점해 **z-점수로
  융합**합니다. GPU 와 CPU 가 동시에 가동되며, Intel GPU+OpenVINO 가 없으면
  CPU 고전 단독으로 자동 폴백합니다. 사진이 많을 때 권장합니다.

### 5.2 강화 전처리 (유사도 "계산 전용")

유사도 계산에만 적용되는 이미지 보정입니다. **썸네일·엑셀에 들어가는
이미지는 원본 그대로 유지됩니다.** 셋업 화면에서 `사진 중앙 30%만 사용
(기준·검증)` 을 켤 수 있습니다.

### 5.3 임계치 슬라이더

- **그룹화 임계치** — 동일 defect 그룹화의 민감도를 조절합니다 (`모든 사진
  자동` 모드의 그룹화 등에 사용).
- **유사도 임계치** — 매칭 기준 점수입니다. 실수 방지를 위해 **마우스 휠로는
  값이 바뀌지 않습니다** (슬라이더를 직접 드래그하세요).

### 5.4 매치 검토 / 결과 검토 / 미매칭 재검토

- **매치 검토 화면** — 대표(최선) 매치 옆에 후보 첫 줄이 함께 표시되며,
  `후보 한 줄 더 보기` 로 후보 줄을 펼치고 `접기` 로 펼친 줄을 **한 번에 모두
  접습니다**(접은 행이 화면 최상단으로 복귀). 각 행의 slot 이름 아래 `크게 보기`
  버튼(또는 기준·매치 사진 우클릭)으로 기준·후보를 **나란히 비교**하며
  `←`/`→` 로 후보를 둘러봅니다. `매치 없음` 을 누르면 해당 행의 후보 썸네일이
  접히고 행 바깥 테두리만 빨간색으로 표시됩니다(되돌리기 가능).
  `[검토 완료]` 시 빨간 표시된 항목이 미매칭으로 처리되고 결과 화면으로
  바로 이동합니다.
- **매칭 결과 검토(결과 화면)** — `[삭제]` 는 즉시 삭제가 아니라 빨간
  테두리로 표시만 합니다. `[확인]` 을 눌러야 실제로 제외됩니다. 제외된
  사진은 ‘매치 실패’ 로 재분류되어, 매치 실패 검토의 **‘매칭 취소 목록’**
  에서 다시 검토할 수 있습니다.
- **매치 실패 사진 검토** — 실패한 기준 사진 목록에서 검토할 사진을 고를 수
  있습니다. 후보는 가로로 **최소 3개씩** 표시되며, 후보/기준 사진을 우클릭
  또는 더블클릭하면 좌우 비교 창에서 **원본 화질** 로 크게 볼 수 있습니다.
  후보를 선택하면 왼쪽 실패 목록에서 해당 항목이 파란색으로 표시됩니다.
  후보 풀이 300장 이상이면 유사도 계산 단계의 점수를 그대로 재사용해 바로
  표시하고, 미만이면 그 자리에서 다시 계산합니다. ‘매칭 취소’ 로 분류된
  항목은 목록 맨 아래 별도 구역에 표시됩니다.

### 5.5 그 밖의 동작

- **모든 새 창(다이얼로그)** 은 최대화(전체 창) 상태로 열립니다.
- **진행 로딩 표시** 는 현재 작업 단계를 `후보 생성` / `유사도 계산` 으로
  표시하고, 진행도를 **처리 갯수(`done / total`)** 로 보여줍니다. 진행 중
  `[중지]` 버튼으로 취소할 수 있습니다.
- **세션 진행 중에는 화면 절전이 방지됩니다.**
- **썸네일 빠른 모드(화질 자동 조정)는 항상 적용됩니다.**
- 검증 결과는 엑셀로 저장됩니다. (매치 결과를 학습 데이터로 누적하던 기능은
  제거되었습니다.)

## 6. 키보드 단축키

| 단축키 | 동작 |
| --- | --- |
| `←` 또는 `1` | 검증 |
| `→` 또는 `2` | 제외 |
| `Z` | 직전 결정 취소 (Undo) |
| `S` | Stage 2 에서 잠시 보류 (Skip) |
| `N` | Stage 2 에서 매칭 없음 확정 |
| `더 크게 보기` 안에서 `←` / `→` | 후보 이전 / 다음 |
| `더 크게 보기` 안에서 `Enter` | 매칭 확정 |
| `더 크게 보기` 안에서 `Esc` | 3-pane 그리드로 복귀 |

### 빠른 모드 / 더 크게 보기

- **빠른 모드(썸네일 화질 자동 조정)는 항상 적용됩니다.** 사진 수에 맞춰 가장
  적절한 화질 티어를 자동으로 선택해 처리 시간을 줄입니다(별도 토글 없음).
- **Stage 2 에서 ‘더 크게 보기’** — 후보 썸네일 우측 상단의 돋보기 버튼을 누르면
  해당 후보를 큰 화면에 펼쳐 보면서 `←` / `→` 로 다른 후보를 둘러보고 `Enter`
  로 매칭을 확정하거나 `Esc` 로 그리드로 돌아갈 수 있습니다. 기존 single-click
  매칭과 double-click 풀스크린 동작은 그대로 유지됩니다.

## 7. 화면 크기 조정

상단 메뉴바 **`화면 크기`** → 5 가지 프리셋 중 선택:

- 작게 (1100×720)
- 보통 (1280×820)  ← 기본
- 크게 (1500×940)
- 매우 크게 (1760×1080)
- 최대화 (모니터 가득)

선택한 크기는 자동으로 저장되어 다음 실행에도 유지됩니다.

검증/매칭 화면 중앙의 사진 크기는 **300 ~ 700 px** 범위에서 슬라이더로
조절 가능합니다 (기본 400 px).

## 8. 캐시 위치

썸네일·중간 크기 이미지·세션 상태는 다음 경로에 저장됩니다.

```
C:\Users\<사용자>\.aoi_verification_cache\
├── thumbs\          ← 200 px 썸네일
├── mid\             ← 800 px 중간 이미지 (엑셀 임베드용)
├── features\        ← 유사도 특징 (.npz)
├── session\         ← 자동 저장 세션
└── ui_prefs.json    ← 화면 크기·임계치·마지막 폴더 등
```

원본 폴더는 절대 수정하지 않습니다. 용량이 부담스러우면 폴더째 지워도 됩니다
(다음 실행 시 자동으로 다시 생성).

## 9. 결과 파일

- 위치: 프로젝트 루트의 `결과/` 폴더 (자동 생성)
- 파일 이름: `AOI {검증호기} 검증 ({기준호기} 기준).xlsx`
- 양식: `양식/양식.xlsx` 를 자동 인식. 헤더 위치(`Slot`, `기준`, `검증`,
  `방향`) 가 자동 감지되므로 어느 열에 두든 동작합니다.
- **미매칭 reference** — Stage 2 에서 ‘잠시 보류’ 또는 ‘매칭 없음 확정’ 처리된
  기준 사진도 같은 워크시트에 한 행으로 함께 들어갑니다.  기준 이미지가
  임베드되고 검증 컬럼에는 빨간 글자로 파일명 텍스트와 ‘미매칭’ 코멘트가
  표시됩니다 (정렬 위치는 매칭 행과 같이 Slot/파일명 순).
