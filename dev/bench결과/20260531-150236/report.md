## 추천: **CPU 임베딩+CPU 융합**
- 이유: 운영 대비 정확도를 보존하면서 가장 빠름.
- 현행(`gpu_fusion_b16`) 대비 속도: **×1.02**
- 연산 방식: GPU/NPU 가 없을 때 CPU(OpenVINO)로 임베딩 추출 후 같은 CPU 가 고전 융합. 가속기 부재 환경의 폴백 성능 측정용.

## 각 레시피의 연산 방식
- **CPU 고전 전수(기준 정확도)** (`cpu_classical_full`): CPU 가 모든 (기준,검증) 쌍을 pHash+ORB+SSIM 으로 전수 비교한다. 임베딩/가속 미사용 — 가장 느리지만 정확도의 '정답 기준선'.
- **GPU 융합 batch16 (현행)** (`gpu_fusion_b16`): Intel GPU(MobileNetV3)가 임베딩을 batch16 으로 뽑아 코사인 후보를 추리고, 상위 40개를 CPU 고전으로 재채점해 z-융합한다. 현행 고효율 모드.
- **NPU 추출+CPU 계산(ResNet18)** (`npu_extract_cpu_fuse`): Intel NPU(ResNet18)가 이미지 임베딩을 '뽑아내고', CPU 가 고전 점수 계산 + z-융합을 맡는다. 사용자 제안 조합(데이터=NPU, 계산=CPU).
- **NPU 추출+CPU 계산(MobileNet)** (`npu_mbnet_cpu_fuse`): NPU 가 GPU 와 동일 백본(MobileNetV3)으로 임베딩을 뽑고 CPU 가 융합. GPU 와 같은 모델이라 분담/대조에 적합.
- **CPU 임베딩+CPU 융합** (`cpu_embed_fusion`): GPU/NPU 가 없을 때 CPU(OpenVINO)로 임베딩 추출 후 같은 CPU 가 고전 융합. 가속기 부재 환경의 폴백 성능 측정용.
- **GPU 임베딩 단독(재채점 없음)** (`gpu_embed_only`): GPU 임베딩 코사인 순위만으로 매칭(CPU 재채점 생략). 가장 빠르지만 정확도가 낮아 '왜 융합이 필요한가'를 보여주는 대조군.
- **GPU+NPU 분담 임베딩+CPU 융합** (`gpu_npu_split_fusion`): 임베딩 작업을 GPU 와 NPU 에 '절반씩 분담'해 동시에 뽑아 추출 처리량을 올리고(중복 아님), CPU 가 융합. 3장치를 속도에 활용하는 핵심 조합.
- **GPU 융합 topk20(얕은 재채점)** (`gpu_fusion_topk20`): GPU 임베딩 후 상위 20개만 CPU 재채점. CPU 단계가 짧아 더 빠르나 정답이 21위 밖이면 놓칠 수 있어 정확도 검증 필수.
- **GPU 융합 topk60(깊은 재채점)** (`gpu_fusion_topk60`): 상위 60개를 CPU 재채점. 더 깊게 보장하나 CPU 비용↑. 정확도 여유가 필요한 어려운 웨이퍼용.
- **GPU 융합+중앙30%crop** (`gpu_fusion_crop`): 고전 재채점을 사진 중앙 30% 로 한정. 호기 간 외곽 차이를 줄여 교차 호기 정확도를 높이는 변형(보고서 권고).
- **GPU 융합 batch1(함정 재현)** (`gpu_fusion_b1`): GPU batch=1 — 보고서상 처리량이 ~1 img/s 로 폭락(멈춤)하는 함정 조합. 운영에서 피해야 함을 수치로 보이는 대조군.
- **GPU 융합 batch4** (`gpu_fusion_b4`): GPU batch=4 — 처리량이 정상화되기 시작하는 지점. batch16 과 속도 비교용.
- **GPU+NPU 앙상블+CPU 융합(대조)** (`gpu_npu_ensemble_fusion`): GPU 와 NPU 가 '각각 전체'를 임베딩해 두 코사인을 평균(앙상블). 보고서상 정확도 이득 0 · 시간 약 2배인 안티패턴 — 분담과 대조.
