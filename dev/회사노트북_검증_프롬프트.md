# 회사 노트북용 분석 프롬프트 — KLA .001(KLARF) 결함 파일 구조

> 회사 노트북(NAS 드라이브 N:/T:/U: 등 접근)에서 KLA 자재의 `.001`(KLARF) 파일 구조를
> 분석합니다. KLA 자재는 `Surface.flt` 가 없고 검사 결과가 `.001` 에 들어 있으므로,
> 미매칭 결함 표기에 쓸 정보(area/size/위치/분류)가 이 파일 어디에 있는지 확인하는 것이
> 목적입니다.
>
> 실행 스크립트는 저장소에 있습니다: **`dev/analyze_kla_001.py`**
> (회사 노트북에 이 repo 가 있으니 그대로 실행. 없으면 그 파일만 복사해도 됩니다.)

---

## 실행 방법

stdlib 만 쓰므로 설치가 필요 없습니다. repo 최신화 후 **그냥 한 줄** 실행하면 분석 대상
4개 파일을 읽어 **단일 마크다운(`AOI_KLA분석.md`) 한 개**로 떨굽니다.

```cmd
python dev/analyze_kla_001.py
```

다른 `.001` 을 보려면 경로를 인자로 줍니다(여러 개 가능):

```cmd
python dev/analyze_kla_001.py "N:\...\A.001" "T:\...\B.001" --out 결과.md
```

옵션: `--max-rows`(표에 보일 레코드 수, 기본 20), `--raw-lines`(원본 verbatim 줄 수, 기본 120).

## 산출 md 에 담기는 것

- **요약표**: 파일별 DefectRecordSpec 컬럼 수 + geometry 후보 컬럼
- 파일별:
  - **헤더** 키-값(FileVersion/LotID/WaferID/StepID/DiePitch/DieOrigin 등)
  - **DefectRecordSpec** — 결함 레코드의 컬럼 정의(여기서 컬럼 이름이 확정됨)
    - 📐 geometry 후보(XSIZE/YSIZE/DEFECTAREA/DSIZE 등) / 📍 위치(XREL/YREL/XINDEX/
      YINDEX) / 🏷 분류(CLASSNUMBER/ROUGHBINNUMBER 등) 자동 표시
  - **DefectList** — 레코드를 컬럼 라벨과 함께 표로(기본 20개) + geometry 컬럼 값 범위
  - **SummaryList** 등 기타 리스트
  - **원본 앞 120줄 verbatim**(파서가 놓친 구조까지 직접 확인용)
