# 회사 노트북용 검증 프롬프트 — Surface.flt geometry vs AOI UI

> 회사 노트북(실제 P:/V:/X:/Y: 드라이브 접근 + AOI UI 확인 가능)에서 Claude Code 에
> **아래 블록을 그대로 붙여넣어** 실행하세요.  실제 `Surface.flt` 에서 뽑은
> area/width/length/**contrast** 가 AOI UI 표시값과 맞는지, 특히 **contrast 가 0 으로
> 나오는 게 정상인지**(Scan Area/zone 63 추정)를 확인하는 것이 목적입니다.

## 붙여넣을 프롬프트 (여기부터)

너는 반도체 AOI 검사 결과 폴더를 분석하는 작업을 한다.  각 결과 폴더의
`Surface.flt`(바이너리, 152바이트 고정 레코드)에서 결함 geometry 를 추출하고, 같은
폴더의 `ColorImageGrabingInfo.ini` 의 좌표로 JPEG 결함과 매칭한 뒤, 그 값이 AOI UI
표시값과 일치하는지 검증하려 한다.

### 확정된 Surface.flt 레코드 스키마 (TB500, 실측 84건으로 검증 완료)

- 레코드는 파일 시작 오프셋 0 부터 **152바이트**씩 연속(헤더 없음). `파일크기 % 152 == 0`.
- 모든 수치는 **little-endian**. 필드별 (레코드 내부 byte offset, 타입):
  - `ActualX` = 24, float64(`<d`)   ← 절대 wafer X (µm). INI 의 FaultX 와 동일.
  - `ActualY` = 32, float64(`<d`)
  - `area` = 80, float64(`<d`)            ← 픽셀² (raw)
  - `BlobBreadth` = 88, float64(`<d`)     ← width 픽셀 (raw)
  - `BlobFeretMax` = 104, float64(`<d`)   ← length 픽셀 (raw)
  - `Contrast` = 136, float64(`<d`)
  - `zone` = 61, uint8(`<B`)
  - `recipe` = 62, uint8(`<B`)
- UI 환산식(TB500, 1px = 0.77µm): `area_um2 = area×0.5929`,
  `width_um = BlobBreadth×0.77`, `length_um = BlobFeretMax×0.77`, `contrast = Contrast(그대로)`.
- 이미지↔레코드 매칭: INI 의 `FaultX/FaultY`(없으면 `X/Y`)와 각 레코드의 `ActualX/ActualY`
  의 유클리드 거리가 최소인 레코드를 고른다. 거리 ≤ 5µm 면 동일 결함으로 본다.

### 할 일

   (이 폴더에는 검증된 예시 2건이 있다: `147206.243725.c.2104939970.2.jpeg`(zone 1, PI Opening,
   contrast≈108.18) 과 `195998.277459.c.1570329500.2.jpeg`(zone 63, Scan Area, contrast=0).)

2. 스크립트 출력 CSV 를 보고, 그 폴더의 결함들을 **zone 별로** 정리한다. 특히 **zone 63
   (Scan Area) 결함의 contrast 가 0 인지**, 다른 zone(예: zone 1, PI Opening)의 contrast 는
   0 이 아닌지 확인한다.

3. **AOI UI 교차 확인(사람이 수행):** 아래 표를 만들어, zone 1 결함 2~3개와 zone 63 결함
   2~3개를 골라 AOI UI 에서 그 결함의 Area/Width/Length/**Contrast** 표시값을 읽어 채운다.
   목표는 다음 두 가지를 확정하는 것:
   - (a) area/width/length 가 UI 와 ±정수 수준에서 일치하는가?
   - (b) **contrast 가 0 인 결함(zone 63 등)에 대해 UI 도 contrast 를 0/공란으로 보여주는가**,
     아니면 UI 는 다른 값을 보여주는가? (후자라면 UI 의 contrast 는 Surface.flt 가 아닌 다른
     소스에서 오는 것이므로 추가 조사 필요.)

   | 파일명 | zone | 추출 area | UI Area | 추출 width | UI Width | 추출 length | UI Length | 추출 contrast | UI Contrast | 일치? |
   |---|---|---|---|---|---|---|---|---|---|---|
   | (스크립트 결과로 채움) | | | | | | | | | | |

4. 최종적으로 **마크다운 보고서** 하나를 만들어:
   - 폴더별 `파일크기 % 152 == 0` 여부(스키마 정합).
   - zone 별 contrast 분포(0 인 비율 vs 0 아닌 비율).
   - 위 UI 교차 확인 표.
   - 결론: "contrast 0 은 zone 63(Scan Area)에서 정상(UI 도 공란)" 인지, 아니면 불일치가 있는지.
   를 정리한다. (이 보고서를 사용자가 복사해 가져갈 것이다.)

5. 시간이 되면 다른 LOT/장비 폴더 2~3개에도 같은 스크립트를 돌려, 위 규칙(특히 contrast/zone)이
   동일하게 성립하는지 본다. (KLA 등 `Surface.flt` 가 없는 폴더는 "측정정보 없음"으로 분류.)

### flt_check.py

```python
import sys, os, re, struct, math, csv
from pathlib import Path
from collections import Counter, defaultdict

REC = 152
F = {  # name: (offset, struct fmt)
    "actual_x": (24, "<d"), "actual_y": (32, "<d"),
    "area": (80, "<d"), "blob_breadth": (88, "<d"),
    "blob_feret_max": (104, "<d"), "contrast": (136, "<d"),
    "zone": (61, "<B"), "recipe": (62, "<B"),
}
AREA_F, LEN_F, TOL = 0.5929, 0.77, 5.0

def parse_flt(p):
    data = Path(p).read_bytes()
    recs = []
    framed_ok = (len(data) % REC == 0)
    pos = 0
    while pos + REC <= len(data):
        r = {}
        for k, (off, fmt) in F.items():
            r[k] = struct.unpack_from(fmt, data, pos + off)[0]
        recs.append(r)
        pos += REC
    return recs, framed_ok, len(data)

def parse_ini(p):
    txt = Path(p).read_text(encoding="utf-8", errors="replace")
    out = {}
    parts = re.split(r"\[([^\]]+)\]", txt)
    it = iter(parts[1:])
    for name, body in zip(it, it):
        kv = dict(re.findall(r"^(\w+)\s*=\s*(.+)$", body, re.M))
        def num(k):
            try: return float(kv[k])
            except: return None
        x = num("FaultX") if num("FaultX") is not None else num("X")
        y = num("FaultY") if num("FaultY") is not None else num("Y")
        if x is not None and y is not None:
            out[Path(name.strip()).stem.lower()] = (x, y, kv.get("Col"), kv.get("Row"),
                                                     kv.get("RecipeNumber"))
    return out

def nearest(recs, x, y):
    best, bd = None, None
    for r in recs:
        d = math.hypot(r["actual_x"] - x, r["actual_y"] - y)
        if bd is None or d < bd:
            bd, best = d, r
    return best, bd

def process(folder, writer, summary):
    folder = Path(folder)
    flt = folder / "Surface.flt"
    ini = None
    for nm in ("ColorImageGrabingInfo.ini", "ColorImageGrabinginfo.ini"):
        if (folder / nm).exists():
            ini = folder / nm; break
    if not flt.exists():
        print(f"[SKIP] Surface.flt 없음(측정정보 미지원): {folder}")
        return
    recs, framed_ok, size = parse_flt(flt)
    print(f"[FLT] {folder}\n      크기={size} (%152=={size % REC}, framed_ok={framed_ok}), "
          f"레코드 {len(recs)}개")
    entries = parse_ini(ini) if ini else {}
    if not entries:
        print("      INI 없음/좌표 없음 — 레코드만 덤프")
        for i, r in enumerate(recs):
            summary[r["zone"]].append(r["contrast"])
            writer.writerow([folder.name, f"rec{i}", r["recipe"], r["zone"], "", "", "",
                             round(r["area"]*AREA_F,3), round(r["blob_breadth"]*LEN_F,3),
                             round(r["blob_feret_max"]*LEN_F,3), round(r["contrast"],3), ""])
        return
    for stem, (x, y, col, row, rn) in entries.items():
        rec, d = nearest(recs, x, y)
        if rec is None or d > TOL:
            writer.writerow([folder.name, stem, rn, "", col, row, round(d or -1,2),
                             "", "", "", "", "측정정보 없음(매칭실패)"])
            continue
        summary[rec["zone"]].append(rec["contrast"])
        writer.writerow([folder.name, stem, rec["recipe"], rec["zone"], col, row, round(d,3),
                         round(rec["area"]*AREA_F,3), round(rec["blob_breadth"]*LEN_F,3),
                         round(rec["blob_feret_max"]*LEN_F,3), round(rec["contrast"],3), ""])

def main():
    roots = sys.argv[1:]
    if not roots:
        print("사용법: python flt_check.py <폴더1> [폴더2 ...]"); return
    out_csv = "flt_check_result.csv"
    summary = defaultdict(list)  # zone -> [contrast...]
    with open(out_csv, "w", newline="", encoding="utf-8-sig") as fp:
        w = csv.writer(fp)
        w.writerow(["folder", "image_stem", "recipe", "zone", "col", "row", "match_dist_um",
                    "area_um2", "width_um", "length_um", "contrast", "note"])
        for root in roots:
            root = Path(root)
            # root 자신 또는 하위에서 Surface.flt 있는 폴더 모두 처리
            targets = [root] if (root / "Surface.flt").exists() else \
                      sorted({p.parent for p in root.rglob("Surface.flt")})
            if not targets:
                print(f"[WARN] Surface.flt 가 없음: {root}")
            for t in targets:
                process(t, w, summary)
    print(f"\nCSV 저장: {out_csv}")
    print("\n=== zone 별 contrast 분포 ===")
    for zone in sorted(summary):
        vals = summary[zone]
        z = sum(1 for v in vals if abs(v) < 1e-9)
        print(f"  zone {zone:>3}: 총 {len(vals):4d}  contrast==0 {z:4d}  "
              f"!=0 {len(vals)-z:4d}  (0 비율 {100*z/len(vals):.0f}%)")
    print("\n해석 가이드: 특정 zone(예: 63=Scan Area)에서 contrast 가 거의 100% 0 이면,")
    print("그 zone 은 원래 contrast 를 측정하지 않는 것(정상). AOI UI 에서도 공란인지 확인할 것.")

if __name__ == "__main__":
    main()
```

## 무엇을 알고 싶은가 (요약)

- area/width/length 는 이미 실측 84건에서 검증 완료(오차 <1e-6). 이번 확인의 핵심은 **contrast**.
- 가설: **contrast=0 은 zone 63(Scan Area) 같은 일부 zone 에서 원래 측정 안 되는 값**이며,
  AOI UI 에서도 그 결함은 contrast 가 공란/0 이다(= 정상). 사용자가 준 예시 2(Scan Area)에서
  실제로 "Contrast 입력 없음"이었던 점이 이를 뒷받침한다.
- 회사 노트북에서 UI 와 대조해 이 가설이 맞으면, 엑셀에서 **contrast 가 0 일 때는 줄을 아예
  빼거나 '—' 로 표기**하도록 바꾸면 깔끔하다(현재는 `contrast 0.00` 으로 보임). 가설이 틀려
  UI 가 0 이 아닌 값을 보여주면, UI contrast 의 진짜 소스를 추가로 찾아야 한다.
