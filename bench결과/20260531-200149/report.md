## 추천: **ORB중앙가중0.5(단독·병렬)**
- 이유: 운영 대비 정확도를 보존하면서 가장 빠름.
- 현행(`gpu_fusion_b16`) 대비 속도: **×4.19**
- 연산 방식: ORB 단독 재채점 + 중앙(defect) 근접 매치에 0.5 가중 — 배경(반복 패턴) 매치 영향을 줄여 defect 판별력↑.  단일 패스(추가 비용 없음).

## 각 레시피의 연산 방식
- **CPU 고전 전수(기준 정확도)** (`cpu_classical_full`): CPU 가 모든 (기준,검증) 쌍을 pHash+ORB+SSIM 으로 전수 비교한다. 임베딩/가속 미사용 — 가장 느리지만 정확도의 '정답 기준선'.
- **GPU 융합 batch16 (현행)** (`gpu_fusion_b16`): Intel GPU(MobileNetV3)가 임베딩을 batch16 으로 뽑아 코사인 후보를 추리고, 상위 40개를 CPU 고전으로 재채점해 z-융합한다. 현행 고효율 모드.
- **고속재채점 병렬(전체)** (`rr_parallel`): 재채점 항목은 그대로(정확도 동일)지만 ref 들을 16스레드로 병렬 채점해 CPU 멀티코어로 시간↓. 정확도 100% 보존하며 속도만↑.
- **CPU재채점 ORB단독** (`cpu_rr_orb_only`): ORB 단독 재채점 — ORB 만의 변별력·비용을 분리해 본다(대조).
- **CPU재채점 pHash+ORB(SSIM뺌)** (`cpu_rr_phash_orb`): SSIM 만 빼고 pHash+ORB. SSIM(전 픽셀 비교) 비용을 없애 속도↑. ORB 가 정확도에 기여하는지 분리 측정.
- **CPU재채점 병렬8(전체)** (`cpu_rr_parallel8`): 전체 항(pHash+ORB+SSIM) 그대로, ref 를 8스레드 병렬 채점 — 정확도 보존하며 멀티코어로 시간↓.
- **CPU재채점 병렬16(전체)** (`cpu_rr_parallel16`): 전체 항 그대로 16스레드 병렬 — 코어 많을수록 이득. 정확도 100% 보존.
- **CPU재채점 병렬32(전체)** (`cpu_rr_parallel32`): 전체 항 그대로 32스레드 병렬 — 과도구독 한계점 확인(코어 수 초과 시 이득 둔화).
- **CPU재채점 ORB특징128** (`cpu_rr_orb128`): ORB 특징 128개 — 더 공격적으로 ORB 비용↓. 정확도 한계 확인용.
- **CPU재채점 ORB특징256** (`cpu_rr_orb256`): 전체 융합이되 ORB 특징을 256개로(기본 500↓) — 검출/정합 비용을 줄여 정확도를 크게 안 깎고 속도↑.
- **고속재채점 ORB+SSIM** (`rr_orb_ssim`): pHash 만 빼고 ORB+SSIM. pHash 영향 분리용 대조.
- **CPU재채점 ORB256+병렬(정확도보존형)** (`cpu_rr_orb256_parallel`): 전체 항을 쓰되 ORB 특징만 256개로 줄이고 16스레드 병렬 — 정확도를 최대한 지키면서 ORB 비용을 깎는 균형형 후보.
- **ORB중앙가중0.5(단독·병렬)** (`rr_orb_center50`): ORB 단독 재채점 + 중앙(defect) 근접 매치에 0.5 가중 — 배경(반복 패턴) 매치 영향을 줄여 defect 판별력↑.  단일 패스(추가 비용 없음).
- **ORB중앙가중0.7(단독·병렬)** (`rr_orb_center70`): ORB 단독 + 중앙 가중 0.7 — defect 비중을 더 키운 변형.
- **융합+ORB중앙가중0.5(병렬)** (`rr_fusion_center50`): 전체 고전(pHash+ORB+SSIM) 융합에서 ORB 항만 중앙 가중 0.5 — 현행 정확도를 유지하며 defect 신호를 강화하는 보수적 변형.
- **NPU보조 defect임베딩 r0.25(3신호)** (`npu_assist_r25`): CPU 재채점과 병행해 NPU 가 상위 후보의 중앙 25%(defect) 임베딩을 batch=1 로 뽑아 3번째 신호로 융합 — 3장치(GPU·CPU·NPU) 동시 활용.
- **NPU보조 defect임베딩 r0.20(3신호)** (`npu_assist_r20`): NPU defect 임베딩 영역을 더 좁혀(20%) defect 에 집중한 변형.
